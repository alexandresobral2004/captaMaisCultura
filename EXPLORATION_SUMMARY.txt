PDF ANALYSIS WORKFLOW
==============================================================================

TRIGGER: Two parallel mechanisms
1. Immediate: POST /api/editais/busca endpoint
2. Scheduled: Background worker (30-min intervals) + weekly full scan

THREE-TIER FALLBACK STRATEGY:
  Tier 1: S3 Pre-signed URL (30% success)
    └─ Direct download from AWS S3
  
  Tier 2: External Link (65% success)
    ├─ If .pdf extension → download directly
    └─ If HTML page → search for PDF inside + extract text
  
  Tier 3: HTML Description (99% success)
    ├─ API-provided description
    ├─ Clean HTML to plain text
    └─ Create mock PDF for records

RESULT: 106 PDFs stored + text extracted from all
SOURCES: 67% HTML, 30% PDF, 3% API descriptions

TOKEN WASTE - CRITICAL ISSUES
==============================================================================

ISSUE #1: 7 Sequential API Calls Per Edital (CRITICAL)
  Location: lib/ai/analyzer.ts
  Problem: Each edital analysis sends full text 7 times
  Cost: 100K tokens per edital × 102 editals = 10.2M already spent
  Waste: Could reduce by 71% with parallel calls
  
  Current calls:
    1. Extract dates (~400 tokens)
    2. Extract values (~400 tokens)
    3. Extract eligibility (~800 tokens)
    4. Extract documents (~600 tokens)
    5. Extract criteria (~600 tokens)
    6. Generate summary (~600 tokens)
    7. Validate consistency (~400 tokens)
  
  Solution: Combine into 3 parallel calls

ISSUE #2: Full Text to Each Prompt (HIGH IMPACT)
  Location: All prompts in analyzer.ts
  Problem: 80,000 chars = ~20,000 input tokens per call
  Solution: Summarize to 10,000 chars before analysis (75% reduction)

ISSUE #3: Classification Before PDF (HIGH IMPACT)
  Location: fetcher.ts + weekly-scan
  Problem: Every edital classified twice before PDF download
  Cost: 600 tokens × 20-30 duplicate checks = 12K-18K tokens/week
  Solution: Cache classification for 30 days

ISSUE #4: Double Validation (Whitelist + OpenAI)
  Location: filtros-ti.ts
  Problem: Whitelist check + OpenAI check both validate IT-relevance
  Cost: 800 tokens × 5-10 items = 4K-8K tokens/week
  Solution: Skip OpenAI if whitelist confidence > 90%

ISSUE #5: HTML Page Analysis
  Location: pdf-downloader.ts
  Problem: Entire web pages extracted as fallback content
  Solution: Limit to first 1000 chars, summarize

==============================================================================
CLASSIFICATION SYSTEM
==============================================================================

THREE-LAYER CLASSIFICATION:

Layer 1: Whitelist/Blacklist (FREE)
  - 140+ technology keywords
  - 130+ institutional keywords (universities, institutes)
  - 35+ blacklist terms (non-TI fields)
  - Result: 80% filter rate
  - Confidence: alta/média/baixa

Layer 2: OpenAI Classification (PAID - 600 tokens)
  - Input: Title, description (500 chars), URL, agency
  - Question: "Is this a REAL edital?"
  - Confidence threshold: >= 70%
  - Fallback: Permissive accept if timeout/error
  - Duplicate calls issue: Happens twice (search + weekly)

Layer 3: Full Content Analysis (PAID - 5-8K tokens)
  - 7 extraction prompts
  - Sets 6 data field groups
  - Sets statusAnalise = "analisado"
  - Sets statusRevisao = "pendente"
  - Awaits human review (not auto-approved)
